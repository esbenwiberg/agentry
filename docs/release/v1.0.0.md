# repofit v1.0.0

Initial public release.

## What's in the box

- **CLI** — `npx @esbenwiberg/repofit` runs the corpus and prints a fitness
  score, dimension breakdown, and per-probe findings.
- **Default corpus** (`@esbenwiberg/corpus-default`, 31 probes across six
  dimensions: Context, Consistency, Cost, Feedback, Latency, Safety).
- **Tiered scheduler** — static → derived → historical → executed →
  reasoned. Executed/reasoned tiers are opt-in via `--include`.
- **Config + baseline** — `repofit --init` writes `repofit.config.json`;
  `repofit --accept` writes `repofit-baseline.json` so CI can ratchet.
- **Gate modes** — `ratchet` (no regression vs baseline), `absolute`
  (per-dimension thresholds), and `advisory` (report-only).
- **Reporters** — human (default), `--json` (full machine output), and
  `--ci` (one-line verdict + GitHub Actions annotations).
- **`repofit explain <probe-id>`** — shows the rationale, evidence, and
  scoring rubric for any probe in the active corpus.
- **SDK** — `defineProbe`, `defineDimension`, and recipes for authoring
  custom probes outside the default corpus.

## Defaults worth knowing

- The CI gate runs the static/derived/historical tiers only. Latency and
  cleanliness probes are opt-in via `npx repofit --include executed` so
  the pipeline stays fast.
- The reading kinds (predicate, count, magnitude, inventory, distribution,
  n/a) and band-based scorer let probes report what they actually saw —
  not a pre-cooked grade.

## Install

```bash
# one-shot
npx @esbenwiberg/repofit

# or add to a repo
npm install --save-dev @esbenwiberg/repofit
npx repofit --init
npx repofit --accept
```
