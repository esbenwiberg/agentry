# repofit v1.1.0

Polyglot release — Node + TypeScript, .NET, Python, and Go are now first-class
supported stacks. Third-party corpora and reporters get explicit extension
points.

## Headline changes

- **Officially supported stacks**: Node + TypeScript, **Python**, **.NET**, **Go**.
  The bundled corpus auto-detects manifests for each (`package.json`,
  `pyproject.toml`, `*.csproj` / `*.sln` / `Directory.Packages.props`,
  `go.mod`) and resolves one argv per phase — build, test, lint, typecheck,
  format — with conservative defaults. Other stacks still get the
  stack-agnostic probes; add stack-specific probes via a custom corpus.
- **Toolchain abstraction**. The 9 executed-tier probes
  (`lint.clean`, `build.clean`, `tests.clean`, `types.clean`, `format.clean`
  and the four `latency.*` probes) and 2 static probes (`types.configured`,
  `readme.commands-runnable`) now run the right command per stack instead
  of hardcoded `npm run X`. All bumped to probe version 2.0.0.
- **Reporter plugin contract**. `defineReporter({ name, render })` mirrors
  `defineProbe`. Wire a third-party reporter via
  `repofit.config.json#reporters.plugins` and invoke it with
  `repofit check --reporter <name>=<path>`.
- **Multi-corpus loading**. `config.corpus` accepts an ordered array; later
  entries override earlier ones on probe id, so a community corpus can
  shadow any stock probe. Override notices print to stderr — never silent.

## Smaller wins

- **JSON Schemas** for `repofit.config.json` and `repofit-baseline.json`
  shipped under `packages/engine/schemas/`. `--init` / `--accept` now write
  the canonical `$schema` URL so editors validate as you type.
- **"No supported stack detected" banner** in the human reporter when a
  repo doesn't match any of the four supported stacks — lists the skipped
  probes and tells the user how to unblock (declare commands or load a
  custom corpus).
- **Per-stack evidence subsystems**: `python_project`, `dotnet_project`
  (with Directory.Packages.props / Central Package Management support),
  `go_module`. Available to corpus authors via the SDK barrel.

## Config additions

```jsonc
{
  "$schema": "https://repofit.dev/schema/config.v1.json",
  "version": 1,
  "corpus": [
    { "package": "@esbenwiberg/corpus-default", "version": "1.1.0" }
    // later entries override earlier ones on probe id
  ],
  "toolchain": {
    "primaryStack": "node",            // optional, defaults to detection
    "commands": {
      "lint": ["sh", "-c", "npm run lint && ruff check ."]  // optional override
    }
  },
  "reporters": {
    "plugins": [
      { "package": "@example/repofit-reporter-csv", "options": { "delimiter": "," } }
    ]
  }
}
```

## Compatibility

- **Probe versions**: 11 probes bumped from 1.0.0 → 2.0.0
  (`*.clean`, `latency.*`, `types.configured`, `readme.commands-runnable`).
  Baselines are keyed by probe id, not version — existing baselines keep
  working. Readings on Node-only repos are unchanged.
- **SDK additions are additive** — no exports removed; no existing types
  changed shape. Custom corpora built against 1.0 keep working.
- Node 22+, TypeScript SDK is ESM (NodeNext), MIT licensed.

## Upgrading

```bash
npm install --save-dev @esbenwiberg/repofit@1.1.0 @esbenwiberg/corpus-default@1.1.0
# re-pin the corpus version in repofit.config.json
```

Re-accepting the baseline is recommended after upgrading — the bumped
probe versions will surface as "version drift" in the report header
until you do.
