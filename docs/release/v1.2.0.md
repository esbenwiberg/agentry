# repofit v1.2.0

Corpus quality release. This adds a broader set of repository-health probes and
fixes the one-shot npm install path for the CLI.

## Headline changes

- **One-shot `npx` works with the default corpus.** `@esbenwiberg/repofit`
  now declares `@esbenwiberg/corpus-default` as a package dependency, so npm's
  temporary install includes the corpus that the CLI loads by default.
- **More agent-readiness probes.** The default corpus now checks machine-readable
  contracts, generated-code marking, local agent skill presence and quality, and
  alignment between agent guidance files.
- **Claude-style guidance forwarding.** Guidance probes now resolve include-only
  lines like `@AGENTS.md`, so a `CLAUDE.md` that forwards to `AGENTS.md` is
  scored on the target content instead of as a one-line stub.
- **Broader pre-commit detection.** The pre-commit probe now recognizes
  `.pre-commit-config.yml`, Lefthook, Overcommit, local `.git/hooks/pre-commit`,
  old Husky package config, and `simple-git-hooks` package config in addition to
  the existing Husky, `.githooks`, and `.pre-commit-config.yaml` paths.
- **Sharper test and architecture signals.** New probes cover architecture
  coupling hotspots, type escape hatches, deterministic tests, test failure
  actionability, spec-to-test traceability, and test oracle quality.
- **Safer default test command detection.** The engine prefers agent-safe test
  commands before falling back to broad project test scripts.

## Upgrading

```bash
npm install --save-dev @esbenwiberg/repofit@1.2.0 @esbenwiberg/corpus-default@1.2.0
```

For one-shot use:

```bash
npx @esbenwiberg/repofit@1.2.0
```

Re-accepting the baseline is recommended after upgrading because the default
corpus version and probe set changed.
